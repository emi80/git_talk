Version control

  • Consistent history
  • Complete history
  • Names
  • Easy access to all version
  • Free playground and easy merge
  • Collaborative

Getting started

  • Identify yourself
  • Create a repository
  • Add files
  • Commit changes

Identify yourself


git config --global user.name "Jack Johnsons"
git config --global user.email "jack@johnsoninc.com"

You do this only once!

Initialize a git project


cd myproject
                    

git init

Branch workflow

  • Create a new branch
  • Switch branches
  • Work on a branch
  • List branches
  • Merge branches
  • git branch <name>
  • git checkout <name>
  • Standard workflow
  • git branch
  • git merge <other_branch>

git checkout -b <name> is a shortcut for create and checkout

Distributed Version Control

Distributed changes

Share repositories and fetch updates

  • No server necessary
  • Clone repositories
  • Pull changes
  • git clone
  • git pull

Central repositories

  • Bare repositories can also be local or remote
  • You can pull and push

Create a bare repository


cd /srv/repos
mkdir myproject; cd myproject
git init --bare --shared
                

cd $HOME/myproject
git remote add origin /srv/repos/myproject
git push -u origin master

Clone a shared repository


git clone /srv/repos/myproject
cd myproject
git remote -v
                

origin	/srv/repos/myproject (fetch)
origin	/srv/repos/myproject (push)

Pull remote changes


git pull origin master

Push changes


vim myscript.py
                

git add myscript.py
                

git commit -m "Fixed some typos"
                

git push origin master
